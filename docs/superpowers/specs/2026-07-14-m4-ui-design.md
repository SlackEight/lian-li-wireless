# M4: UI — Design

**Date:** 2026-07-14
**Status:** Approved (design walked through with owner in two parts + three visual decisions via mockup session; mockups in `.superpowers/brainstorm/48378-*/content/`, gitignored but described below)

## 1. Goal

The L-Connect-class control surface: a Tauri 2 desktop app over the existing daemon, plus the bind/unbind capability that removes the last dependency on any other software for device onboarding.

## 2. Locked visual decisions (owner-selected from rendered mockups)

1. **Design language: "Nocturne Charged."** Refined dark (near-black neutrals `#0b0b0f`/`#141419`, hairline borders `rgba(255,255,255,0.07)`, quiet system-font typography) where the only saturated color is the devices' actual RGB — and that RGB *blooms*: soft glow bleeding off color-bearing elements (fan rings `box-shadow` bloom, lit chip edges, glowing accent bars). The app frames the light show. Reference card: mockup `visual-style-v2.html` (charged variant).
2. **Shell: section sidebar.** Fixed left sidebar — Lighting / Cooling / Devices / Health — content fills the rest. Devices appear inside Lighting/Cooling as pickers.
3. **Lighting screen: Stage.** Big live device canvas as hero (devices drawn with their REAL layout-map geometry — SL-INF's inner ring, arcs, and side strips as they physically are), right rail with effect list + palette + speed/direction/brightness controls, preset chips under the stage.

Plus: **plain window** (no tray, no autostart — the daemon is the resident part), and **bind/unbind is IN scope** (owner chose the ambitious option).

## 3. Architecture

- **`crates/llw-ui`** — Tauri 2, React 19 + Vite frontend (switched from Svelte 5 at owner request during M4b Task 2 — see the M4b plan). Rust side is a thin IPC client to the daemon socket (envelope v1) exposed as Tauri commands; zero policy.
- **Previews via WASM:** `llw-effects` (pure, serde-only) compiles to WASM; the webview renders the Stage canvas and effect thumbnails at 60fps locally with the exact code that drives hardware. A parity test pins WASM output == native goldens.
- **State:** UI polls `Status` (~1s while window focused); all writes via existing/new IPC requests. Daemon-unreachable → cached read-only render + reconnect banner (spec §7 inherited). The 3s RF settle window is surfaced honestly: Apply shows an "applying — RF quiet window" state until `rgb_sync` confirms.

## 4. Screens

- **Lighting (Stage):** live canvas; effect rail; palette editor (≤8 colors); speed/direction/brightness; instant local preview, explicit Apply → `SetEffect`; preset chips (presets = named EffectSpecs, new config section + IPC).
- **Cooling:** drag-point curve editor with live temp cursor; sensor picker (hwmon enumeration via a new `ListSensors` IPC); per-slot assignment; live RPM/PWM. Writes via `SetConfig`.
- **Devices:** the air inventory — bound-to-us (rename, info, unbind-with-confirm), unbound (Bind button), bound-elsewhere (shown, bind refused unless forced — this refusal IS the "bind lock", implemented as policy; upstream has no wire-level lock).
- **Health:** link/channel, reliability telemetry, wedge banner, per-device sync + dropout streaks.

## 5. Bind/unbind (daemon work, M4a — precedes all UI)

Protocol (from upstream bind.rs @ d262007, verified): bind = PWM-shaped frame with `rf[8..14]=target_master`, `rf[14]=rf[16]=target_rx` (an unused endpoint 1–14), sent 6×@30ms; convergence verified by GetDev polls (≤5s, 150ms cadence); then SaveConfig broadcast (`rf[1]=0x15`, dst ff:ff:ff:ff:ff:ff, rx 0xFF, 3×@200ms) persists to device flash. Unbind = same with zero master + rx 0.

Our shape: pure frame builders in `llw-protocol` (byte-tested); burst/save methods on `Dongle`; **convergence is supervisor policy** — `Bind{mac}` IPC replies `started` immediately, the supervisor runs the burst + tracks a pending-bind through its normal step polls until converged/deadline, Status reflects the outcome (UI polls anyway; no long-blocking IPC). Post-bind: device auto-enters config (existing first curve, else 40% constant; no RGB). SaveConfig is followed by the RF settle window (rename `rgb_settle_until` → `rf_settle_until`). Air inventory: supervisor tracks ALL records with bound-to-us / foreign / unbound classification.

## 6. Testing

Bind frames byte-tested vs upstream; supervisor bind/unbind/refusal flows FakeIo-simulated; WASM/native render parity test; curve-editor math in Vitest; per-milestone owner acceptance (visuals judged against the locked mockups); `llw bind`/`llw unbind` CLI lands with M4a for hardware validation before any UI exists.

## 7. Delivery

M4a bind+inventory+IPC+CLI → M4b Tauri shell + Health + Devices → M4c Lighting stage + WASM previews + presets → M4d Cooling editor → M4e polish + acceptance. AUR packaging of the app remains M5.

## 8. Risks

Bind touches other people's device state — refusal-by-default on foreign masters, confirm-gated unbind, and hardware validation gated on the owner's own spare... the owner's devices only (SL-INF cluster is bound; the acid test is binding the STRIMER when installed — which doubles as its deferred M3 validation). WASM toolchain (wasm-pack/wasm-bindgen) is new to the workspace — isolated to llw-effects which has no native deps, low risk. Tauri 2 on KDE Wayland: webkit2gtk quirks possible; mitigation is early M4b smoke on the owner's machine.
